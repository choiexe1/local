# 출석 관리 앱
**출석 관리 앱 (Attendance Manager)**, 더 구체적인 기획과 설계를 단계별로 설명해줄게. 이 앱은 학원 운영의 출석 관리 효율성을 높이는 데 초점을 맞추고, QR 코드 스캔과 버튼 입력을 통해 간단하면서도 실용적으로 만들어볼게.

---

### 앱 개요

- **목표**: 학생 출석을 빠르고 정확하게 기록하고, 날짜별/학생별로 데이터를 조회할 수 있는 앱.
- **대상 사용자**: 학원 강사나 운영자.
- **핵심 기능**: QR 코드로 출석 체크, 수동 버튼 입력, 데이터 저장 및 조회.
- **제한 조건**: 일주일 안에 완성 가능한 수준으로 설계 (기본 기능 위주, 확장 가능성 열어둠).

---

### 기능 상세 설계

#### 1. **메인 화면 (출석 체크 화면)**

- **UI 구성**:
    - 상단: 학원 이름 또는 "오늘의 출석" 타이틀.
    - 중앙: 학생 리스트 (이름만 간단히 표시).
    - 각 학생 옆: "출석" 버튼 (탭하면 출석으로 기록).
    - 하단: "QR 스캔" 버튼 (카메라 열어서 QR 코드 인식).
- **동작**:
    - "출석" 버튼을 누르면 해당 학생의 출석 여부가 즉시 기록됨.
    - "QR 스캔" 버튼을 누르면 QR 코드 스캔 화면으로 이동.
- **추가 고려**:
    - 날짜 선택 가능 (기본은 오늘 날짜로 설정).

#### 2. **QR 코드 스캔 기능**

- **UI 구성**:
    - 카메라 뷰와 "스캔 중" 텍스트.
    - 스캔 성공 시 팝업: "학생 [이름] 출석 완료".
- **동작**:
    - 학생마다 고유 QR 코드를 미리 생성 (예: 학생 ID 기반).
    - QR 코드 스캔 시 해당 학생의 출석을 자동 기록.
- **구현 포인트**:
    - qr_code_scanner 같은 패키지 사용.
    - QR 코드 데이터는 학생 이름이나 ID로 단순화.

#### 3. **출석 조회 화면**

- **UI 구성**:
    - 탭으로 전환 가능한 두 개의 뷰:
        1. **날짜별 조회**: 날짜를 선택하면 그날 출석한 학생 리스트 표시.
        2. **학생별 조회**: 학생을 선택하면 그 학생의 최근 출석 기록 표시.
    - 간단한 테이블 또는 리스트 형태로 출력.
- **동작**:
    - 로컬 데이터베이스에서 데이터를 불러와 표시.
    - 필터링 기능 (날짜/학생 선택).

#### 4. **데이터 관리**

- **저장 방식**:
    - 로컬 데이터베이스(SQFlite) 사용.
    - 테이블 구조 예시:
        
        text
        
        접기자동 줄바꿈복사
        
        `Table: Attendance - id (자동 증가) - student_name (텍스트) - date (날짜, 예: "2025-04-09") - is_present (불리언, 출석 여부)`
        
- **데이터 입력**:
    - 출석 버튼 또는 QR 스캔 시 자동 저장.
- **데이터 조회**:
    - 날짜별: SELECT * FROM Attendance WHERE date = ?
    - 학생별: SELECT * FROM Attendance WHERE student_name = ?

---

### 개발 흐름 (일주일 계획)

일주일 안에 끝내려면 아래처럼 작업을 나눠서 진행하면 좋아.

- **Day 1: 환경 설정 및 기본 UI**
    - Flutter 프로젝트 생성, 패키지 추가 (sqflite, qr_code_scanner 등).
    - 메인 화면 UI 설계 (학생 리스트, 버튼).
- **Day 2: 데이터베이스 설정**
    - SQFlite로 테이블 생성 및 CRUD 함수 구현.
    - 더미 데이터로 테스트.
- **Day 3: 출석 체크 기능**
    - 버튼으로 출석 기록 기능 완성.
    - 데이터베이스와 연동.
- **Day 4: QR 코드 스캔**
    - QR 스캔 패키지 연동, 학생 ID 기반 출석 기록 구현.
    - 스캔 성공 시 팝업 추가.
- **Day 5: 조회 화면**
    - 날짜별/학생별 조회 UI 설계.
    - 데이터베이스 쿼리로 데이터 표시.
- **Day 6: 테스트 및 디버깅**
    - 더미 학생 데이터로 전체 흐름 테스트.
    - UI/UX 개선 (버튼 색상, 간격 조정 등).
- **Day 7: 마무리**
    - 최종 점검 후 APK 빌드.
    - 추가하고 싶은 간단한 기능(예: 출석률 계산) 구현.

---

### 기술적 세부사항

1. **패키지 추천**:
    - sqflite: 로컬 데이터베이스.
    - qr_code_scanner: QR 코드 스캔.
    - provider 또는 get: 상태 관리 (선택).
2. **QR 코드 생성**:
    - 앱 외부에서 학생별 QR 코드를 미리 만들어야 함 (예: 학생 ID를 텍스트로 인코딩).
    - 앱 내에서 생성하려면 qr_flutter 패키지 추가 가능.
3. **UI 스타일**:
    - 심플하게: Material Design 활용.
    - 색상은 학원 느낌에 맞춰 파란색/녹색 계열 추천.

---

### 확장 가능성

일주일 후에 더 발전시키고 싶다면:

- **출석률 통계**: 학생별 출석 퍼센트 표시.
- **클라우드 연동**: Firebase로 데이터 백업.
- **학생 추가/삭제**: 관리자용 학생 명단 편집 기능.

---

### 예상 결과물

- 강사가 앱을 켜고 학생 이름 옆 버튼을 누르거나 QR 코드를 스캔하면 출석이 기록됨.
- "조회" 탭에서 지난 출석 기록을 쉽게 확인 가능.
- 학원에서 수기 출석부를 대체할 수 있는 실용적인 도구 완성!

# 랜덤 운명 상자
좋았어! **랜덤 운명 상자 (Fate Box)**를 선택했으니, 이 아이디어를 구체적으로 기획해서 백엔드 없이 로컬로 구현 가능하고, 일주일 안에 완성할 수 있는 플러터 프로젝트로 만들어줄게. 앱의 재미와 독창성을 살리면서도 개발 난이도를 적당히 유지할게!

---

### 앱 개요

- **목표**: 사용자가 선택지를 입력하면 상자가 흔들리는 애니메이션 후 랜덤으로 결과를 뽑아주는 앱. 과거 뽑기 기록도 저장해서 확인 가능.
- **대상 사용자**: 결정을 못 내리는 사람, 재미있는 경험을 원하는 누구나.
- **핵심 기능**: 선택지 입력, 랜덤 뽑기 애니메이션, 기록 저장 및 조회.
- **기발한 요소**: 상자가 흔들리는 극적인 애니메이션과 "운명" 콘셉트로 몰입감 추가.
- **제한 조건**: 로컬 데이터 사용, 일주일 개발 기간.

---

### 기능 상세 설계

#### 1. **선택지 입력 화면**

- **UI 구성**:
    - 상단: "운명을 맡겨보세요" 타이틀.
    - 중앙: 선택지 입력란 (최소 2개, 최대 10개).
        - 각 줄에 TextField로 입력 가능.
        - "+" 버튼으로 입력란 추가, "-" 버튼으로 삭제.
    - 하단: "상자에 넣기" 버튼.
- **동작**:
    - 사용자가 최소 2개 이상 선택지를 입력해야 버튼 활성화.
    - 버튼 누르면 다음 화면(뽑기 화면)으로 이동.
- **구현 포인트**:
    - ListView나 Column으로 동적 입력란 관리.
    - 입력값 유효성 검사 (빈칸 체크).

#### 2. **랜덤 뽑기 화면**

- **UI 구성**:
    - 중앙: 상자 이미지 (기본은 닫힌 상태).
    - 하단: "흔들기" 버튼 (또는 "운명 결정" 텍스트).
    - 결과 표시: 상자 열리며 선택지 하나 팝업으로 등장.
- **동작**:
    - "흔들기" 버튼 누르면 상자가 좌우로 흔들리는 애니메이션 시작 (약 2-3초).
    - 애니메이션 끝나면 상자 열리고, 랜덤으로 뽑힌 결과 표시.
    - "저장" 버튼으로 기록 남기기 가능.
- **구현 포인트**:
    - AnimatedContainer나 AnimationController로 흔들기 효과.
    - Dart의 Random으로 결과 선택.
    - 결과는 Dialog나 화면 전환으로 표시.

#### 3. **운명 기록 조회 화면**

- **UI 구성**:
    - 상단: "운명의 서" 타이틀.
    - 중앙: 날짜별 뽑기 기록 리스트 (예: "2025-04-09: 치킨").
    - 클릭 시: 해당 뽑기의 선택지 전체와 결과 표시.
- **동작**:
    - 로컬 DB에서 과거 기록 불러와 리스트로 출력.
    - 기록 선택 시 세부 정보 확인 가능.
- **구현 포인트**:
    - ListView로 기록 표시.
    - 간단한 팝업으로 세부 내용 보여주기.

#### 4. **데이터 관리**

- **저장 방식**:
    - 로컬 데이터베이스(SQFlite) 사용.
    - 테이블 구조 예시:
        
        text
        
        접기자동 줄바꿈복사
        
        `Table: FateRecords - id (자동 증가) - date (날짜, 예: "2025-04-09") - options (텍스트, JSON으로 선택지 리스트 저장) - result (텍스트, 뽑힌 결과)`
        
- **데이터 입력**:
    - 뽑기 완료 후 "저장" 시 DB에 기록.
- **데이터 조회**:
    - SELECT * FROM FateRecords ORDER BY date DESC로 최신순 조회.

---

### 개발 흐름 (일주일 계획)

- **Day 1: 환경 설정 및 입력 UI**
    - Flutter 프로젝트 생성, sqflite 패키지 추가.
    - 선택지 입력 화면 설계 및 동적 입력란 구현.
- **Day 2: 뽑기 화면 기본**
    - 상자 이미지와 버튼 UI 설계.
    - 랜덤 선택 로직 추가.
- **Day 3: 애니메이션**
    - 상자 흔들기 애니메이션 구현.
    - 결과 표시 UI 완성.
- **Day 4: 데이터 저장**
    - SQFlite로 기록 저장 기능 구현.
    - 더미 데이터로 테스트.
- **Day 5: 기록 조회**
    - 조회 화면 UI 설계.
    - DB에서 데이터 불러와 표시.
- **Day 6: 테스트 및 개선**
    - 전체 흐름 점검 (입력 → 뽑기 → 저장 → 조회).
    - UI 조정 (색상, 애니메이션 속도 등).
- **Day 7: 마무리**
    - 최종 디버깅 후 APK 빌드.

---

### 기술적 세부사항

1. **패키지**:
    - sqflite: 로컬 데이터 저장.
    - (선택) flutter_spinkit: 흔들기 전 로딩 효과 추가 가능.
2. **애니메이션**:
    - Transform.translate로 좌우 흔들기 구현.
    - 상자 열리는 효과는 투명도 조절(Opacity)로 간단히.
3. **UI 스타일**:
    - 상자는 신비로운 보물상자 느낌 (금색 테두리, 나무 질감).
    - 배경은 어두운 톤으로 운명의 분위기 강조.
